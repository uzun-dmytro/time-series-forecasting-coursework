# Об'єкти дослідження (Домени)
## Основні (Практичні) домени:
1. Енергетика (споживання): Змішана система. Присутня фізична залежність (погода) та сильні соціальні/календарні цикли. Очікуваний лідер: XGBoost.
2. Фінанси (S&P 500): Високозашумлена система, випадкове блукання. Прогнозується не абсолютна ціна, а напрямок (Directional Accuracy). Очікуваний лідер: SVR з жорсткою регуляризацією.
3. Метеорологія (Jena Climate): Чиста фізична система зі строгими межами, високою інерцією та ідеальною автокореляцією. Очікуваний лідер: класична статистика (ARIMA/SARIMA) або XGBoost (за наявності екзогенних факторів).
4. Якість повітря (Beijing PM2.5): Зашумлена система з екзогенною залежністю від метеоумов (вітер, тиск, температура, опади). Характерні різкі стрибки (епізоди смогу) та добова/сезонна циклічність. Очікуваний лідер: XGBoost або LSTM.
5. Інші домени будуть додані в майбутньому.

## Додаткові (Теоретичні) домени для порівняння в тексті:
- Рітейл (Роздрібні продажі): Переривчастий попит (intermittent demand), де багато нулів, яскраво виражена річна сезонність.
- Медичні дані (ЕКГ, пульс): Високочастотні дані з мікропатернами, де глибоке навчання розкриває свій максимальний потенціал.

# Перелік моделей для тестування (Алгоритмічний арсенал)
## 1. Базові моделі (Baselines)
- Naive Forecast (t-1): Прогноз на основі попереднього значення (фізична інерція).
- Seasonal Naive (t-m): Прогноз на основі попереднього сезону.

## 2. Класичні статистичні моделі
- ARIMA: Для рядів без чіткої сезонності (у фінансах зводиться до ARMA, $d=0$).
- SARIMA: Розширення для даних із чітким циклом.
- Holt-Winters (Експоненційне згладжування): Швидка альтернатива SARIMA.

## 3. Моделі машинного навчання (ML)
- XGBoost: Градієнтний бустинг над деревами рішень (вбудований відбір ознак).
- SVR: Метод опорних векторів із радіальною базисною функцією (RBF).

## 4. Моделі глибокого навчання (DL)
- MLP: Багатошаровий перцептрон (базовий DL-бенчмарк).
- LSTM: Рекурентна архітектура для довгострокових залежностей.
- 1D-CNN: Згорткова нейромережа для виділення локальних патернів.

# Процес розробки та оптимізації (Двоетапний підхід)
Для забезпечення прозорості експериментів розробка алгоритмів завжди повинна відбуватися у два етапи:

## Етап 1: Базові конфігурації (Standard Versions)
- Усі моделі генеруються з параметрами за замовчуванням (або з мінімально необхідними для запуску).
- Мета: отримати чистий baseline продуктивності алгоритму "з коробки".

## Етап 2: Глибока оптимізація (Тільки за явним запитом)
- Застосовується виключно після прямої команди користувача.
- DL: Впровадження `EarlyStopping`, `ReduceLROnPlateau`, `BatchNormalization`, `Dropout`.
- ML: Пошук гіперпараметрів (GridSearchCV/Optuna), налаштування регуляризації SVR ($C$, $\gamma$, $\epsilon$) та XGBoost.
- Виняток: для фінансового домену (S&P 500) Optuna не застосовується, оскільки ряд є випадковим блуканням і додаткова оптимізація гіперпараметрів не дає статистично значущого покращення. Це усвідомлене рішення, підтверджене експериментально.
- Статистика: Мінімізація інформаційних критеріїв (AIC/BIC), тест Льюнга-Бокса.
- Валідація: Суворий хронологічний спліт (TimeSeriesSplit), заборона на випадковий K-Fold.

# Суворі правила генерації (System Rules)
1. Абсолютна заборона на використання емодзі в коді, коментарях або академічному тексті.
2. Текст курсової має бути написаний формальною українською науковою мовою (використовувати пасивний стан або першу особу множини "Ми дослідили...").
3. Математичні формули формувати виключно у форматі LaTeX (використовувати `$` для вбудованих та `$$` для виділених блоків). Не використовувати просторових відступів між знаком долара та формулою.
4. Метрики мають бути контекстними: для фінансів — Directional Accuracy (DA) та MAPE для наївного прогнозу; для метеорології та якості повітря — MAE та RMSE.
5. Аналітика має фокусуватися на поясненні "ЧОМУ" (прокляття розмірності, витік даних, лінійна автокореляція), а не на сухій констатації цифр.
6. Завжди дотримуватися принципу: "Складніше не означає краще". Перевага віддається найпростішій моделі, яка вирішує задачу.
